# Databases and views

A database is one set of rows with typed columns — text, select, date, number, checkbox and more. Views are different faces on the same rows: a table for editing, a board for flow, a gallery for browsing, a calendar for time, a habit grid for streaks.

<!-- otion:info {"color":"green","icon":"Table","text":"The **First Steps** database on your Home page has both a table and a board view — switch between them with the tabs above it."} -->

## Things worth knowing

- Click a column header to sort; drag rows to reorder.
- Rows can nest: drag a row onto another to make it a sub-item.
- A number column can compute itself — set it to **sum**, **avg**, **min** or **max** and it aggregates its sub-items. The "Explore databases" row on your Home page does exactly this.
- Every database lives in `databases/` as a single readable JSON file.

## Make your own

Hover over **Databases** in the sidebar and press **+**, or import a CSV/Excel file you already have.
